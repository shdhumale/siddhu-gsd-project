# Plan 01-01: Spring Boot & MySQL Foundation

Establish the backend infrastructure and data persistence layer.

<tasks>
<task id="1" name="Initialize Spring Boot Project">
<read_first>
- .planning/phases/01-foundation/01-RESEARCH.md
- .planning/phases/01-foundation/01-CONTEXT.md
</read_first>
<action>
Create a new Spring Boot 3.4 project in `/ticket-system-backend` using Maven.
Include dependencies:
- `spring-boot-starter-web`
- `spring-boot-starter-data-jpa`
- `spring-boot-starter-validation`
- `mysql-connector-j`
- `springdoc-openapi-starter-webmvc-ui` (v2.7.0)
</action>
<acceptance_criteria>
- Directory `ticket-system-backend/` exists.
- `pom.xml` contains all specified dependencies.
- `./mvnw clean compile` succeeds.
</acceptance_criteria>
</task>

<task id="2" name="Configure MySQL Connection">
<read_first>
- ticket-system-backend/src/main/resources/application.properties
</read_first>
<action>
Set up MySQL connection properties in `application.properties`:
- `spring.datasource.url=jdbc:mysql://localhost:3306/ticket_db?createDatabaseIfNotExist=true`
- `spring.datasource.username=root`
- `spring.datasource.password=` (or user discretion)
- `spring.jpa.hibernate.ddl-auto=update`
- `spring.jpa.show-sql=true`
</action>
<acceptance_criteria>
- `application.properties` contains valid MySQL configuration.
</acceptance_criteria>
</task>

<task id="3" name="Implement Ticket Entity & Repository">
<read_first>
- .planning/REQUIREMENTS.md ([REQ-005])
</read_first>
<action>
Create `Ticket.java` entity with fields:
- `Long id` (Generated, PK)
- `String name`
- `String description`
- `String status`
Create `TicketRepository.java` extending `JpaRepository<Ticket, Long>`.
</action>
<acceptance_criteria>
- `Ticket.java` has correct JPA annotations (@Entity, @Id).
- `TicketRepository.java` exists.
</acceptance_criteria>
</task>

<task id="4" name="Seed Mock Data">
<read_first>
- .planning/phases/01-foundation/01-CONTEXT.md (D-03)
</read_first>
<action>
Implement `TicketBootstrap.java` using `CommandLineRunner`.
Upon startup, if repository is empty, save at least 5 mock tickets with varied statuses (New, Assigned, Done).

<acceptance_criteria>

</task>
</tasks>
